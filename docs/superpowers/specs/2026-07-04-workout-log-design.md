# 운동 기록 사이트 설계 (2026-07-04)

## 목적

개인용 운동 기록 웹사이트. 헬스장에서 휴대폰으로 접속해 종목·무게·횟수·세트를 빠르게 입력하고, 기록은 구글 시트에 쌓인다. 완전 무료로 호스팅한다.

## 전체 구조

- **프론트엔드**: 순수 HTML/CSS/JS 정적 사이트 (프레임워크 없음). GitHub Pages에 무료 호스팅. PWA 매니페스트를 포함해 휴대폰 홈 화면에 앱처럼 추가할 수 있다.
- **백엔드**: 구글 Apps Script 웹앱 1개. 구글 시트에 읽기/쓰기하는 API 역할. 무료, 서버 관리 없음.
- **데이터 저장소**: 구글 시트 1개. 시트에서 직접 열어 보거나 수정해도 사이트와 그대로 연동된다.
- **저장소/배포**: 이 폴더가 git 저장소가 되고 GitHub에 push → GitHub Pages로 서빙. GitHub 저장소 이름은 `gym-log` (Pages URL: `https://<username>.github.io/gym-log/`).

## 데이터 모델 (구글 시트)

시트 이름: `기록`. 1행 = 세트 1개.

| 열 | 이름 | 예시 |
|---|---|---|
| A | 날짜 | 2026-07-04 |
| B | 종목 | 벤치프레스 |
| C | 무게(kg) | 60 |
| D | 횟수 | 10 |
| E | 세트번호 | 1 |
| F | 기록ID | 클라이언트 생성 고유 문자열 (삭제·중복방지용) |

- 날짜는 `YYYY-MM-DD` 문자열로 저장 (시간대 혼동 방지, Asia/Seoul 기준).
- 기록ID는 클라이언트가 생성한 고유 문자열. 오프라인 재전송 시 같은 ID면 중복 저장하지 않는다(멱등성).
- 맨몸 운동은 무게 0 허용.

## API (Apps Script 웹앱)

- `doGet` — 조회. `action` 파라미터로 분기:
  - `today`: 오늘 날짜의 기록 목록
  - `history`: 날짜별 과거 기록 (최근 N일)
  - `last?exercise=벤치프레스`: 해당 종목의 가장 최근 날짜 기록 (지난번 무게 자동 표시용)
  - `exercises`: 최근 사용한 종목 목록 (최근 사용순)
  - `chart?exercise=벤치프레스`: 날짜별 최고 무게 시계열
- `doPost` — 변경. JSON body의 `action`으로 분기:
  - `add`: 세트 추가 (기록ID로 멱등 처리)
  - `delete`: 기록ID로 세트 삭제
- 모든 요청에 비밀 토큰(`token`) 필수. Apps Script의 스크립트 속성에 저장된 값과 대조하고, 불일치 시 거부. 프론트에는 최초 접속 시 토큰을 입력받아 localStorage에 저장한다 (GitHub 공개 저장소에 토큰이 노출되지 않게 하드코딩 금지).
- CORS: Apps Script 제약 때문에 POST는 `Content-Type: text/plain`으로 JSON을 보내고, 응답은 `ContentService` JSON. 리다이렉트는 fetch 기본 동작(follow)으로 처리.

## 화면 (모바일 1페이지, 탭 3개)

### 탭 1: 오늘 (기본 화면)
1. **종목 선택**: 최근 쓴 종목이 버튼 목록으로 표시 — 탭 한 번에 선택. 새 종목은 텍스트로 직접 입력.
2. **지난번 기록 표시**: 종목 선택 시 "지난번: 60kg × 10 × 3세트"가 폼 위에 표시되고, 무게/횟수 입력칸이 그 값으로 자동 채워진다.
3. **입력 폼**: 무게(kg)·횟수 숫자 입력(`inputmode="decimal"/"numeric"` 로 모바일 숫자 키패드) + [세트 추가] 버튼. 같은 종목 연속 입력 시 값이 유지되어 버튼만 누르면 다음 세트가 기록된다. 세트번호는 자동 증가.
4. **오늘 기록 목록**: 오늘 입력한 세트가 종목별로 묶여 표시. 각 세트에 삭제 버튼.

### 탭 2: 지난 기록
- 날짜별로 접었다 펴는 목록. 최근 날짜부터. 종목별로 묶어 "벤치프레스 60kg×10, 60kg×8 …" 형태로 표시.

### 탭 3: 차트
- 종목 선택 드롭다운 → 날짜별 최고 무게 꺾은선 그래프. 외부 라이브러리 없이 SVG로 직접 렌더링.

## 오프라인/에러 처리

- 세트 추가 시 **낙관적 UI**: 즉시 화면에 반영하고 백그라운드로 전송.
- 전송 실패(네트워크 불안정) 시 해당 세트를 localStorage 대기열에 저장하고 "전송 대기 중" 배지 표시. 앱 재접속·재시도 시 대기열을 자동 재전송한다. 기록ID 멱등성 덕분에 중복 걱정 없음.
- 조회 실패 시 마지막으로 성공한 응답을 localStorage 캐시에서 보여주고 "오프라인 데이터" 표시.

## 테스트

- Apps Script 쪽: 시트 조작 함수를 순수 함수(파싱·필터링·멱등 체크)와 시트 I/O로 분리하고, 순수 함수는 로컬에서 Node로 단위 테스트.
- 프론트 쪽: 데이터 가공 로직(종목별 그룹핑, 차트 시계열 변환, 대기열 처리)을 별도 모듈로 분리해 Node 단위 테스트. UI는 브라우저에서 수동 확인.

## 배포 (최초 1회 세팅)

1. 구글 시트 생성 → 확장 프로그램 > Apps Script에 제공된 코드 붙여넣기 → 스크립트 속성에 토큰 설정 → 웹앱으로 배포(액세스: 모든 사용자) → 웹앱 URL 확보.
2. 프론트 설정 화면(또는 최초 접속 프롬프트)에 웹앱 URL과 토큰 입력 → localStorage 저장.
3. GitHub에 `gym-log` 저장소 생성 → push → Settings에서 Pages 활성화.
4. 휴대폰에서 접속해 "홈 화면에 추가".

## 하지 않는 것 (YAGNI)

- 로그인/멀티유저 — 개인용, 토큰 하나로 충분.
- 운동 프로그램/루틴 추천, 타이머, 칼로리 계산.
- 프레임워크·빌드 도구 — 정적 파일 그대로 서빙.
